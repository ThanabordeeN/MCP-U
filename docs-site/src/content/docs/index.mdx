## Why MCP/U?

<CardGrid stagger>
  <Card title="Self-Describing Firmware" icon="document">
    Devices auto-declare their tools via JSON Schema. The client registers tools dynamically вҖ” zero hardcoded names.
  </Card>
  <Card title="Universal Transport" icon="random">
    Serial, WiFi (TCP), and Bluetooth вҖ” one library, one API, any Arduino `Stream`.
  </Card>
  <Card title="Minimalist Integration" icon="pencil">
    Just `add_pin()`, `add_tool()`, `begin()`. No protocol complexity in your sketch.
  </Card>
  <Card title="Multi-Device Ready" icon="laptop">
    Connect any number of MCUs to a single MCP client. Tools auto-namespaced per device.
  </Card>
</CardGrid>

## How It Works

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ        в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ        в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Claude / LLM   в”Ӯв—„в”ҖMCPв”Җв”Җв–әв”Ӯ  mcpu-client (npm)   в”Ӯв—„в”Җв”Җв”Җв”Җв”Җв–әв”Ӯ  MCU Firmware в”Ӯ
в”Ӯ                 в”Ӯ  stdio в”Ӯ  Dynamic tool regist. в”Ӯ Serial в”Ӯ  McpIot lib   в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ        в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ   /TCP в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

1. Client connects to MCU over Serial or TCP
2. Calls `list_tools` вҖ” MCU responds with full tool + pin registry (JSON Schema)
3. Client dynamically registers one MCP tool per MCU tool
4. Claude can now call any MCU tool by name

## 3-Step Setup

<Tabs>
  <TabItem label="1 вҖ” Flash Firmware">
```cpp
#include <McpIot.h>

McpDevice mcp("my-device", "1.0.0");

void setup() {
  mcp.add_pin(2,  "led",    MCP_DIGITAL_OUTPUT, "Onboard LED");
  mcp.add_pin(34, "sensor", MCP_ADC_INPUT,      "Analog Sensor");
  mcp.begin(Serial, 115200);
}

void loop() { mcp.loop(); }
```
  </TabItem>
  <TabItem label="2 вҖ” Add to Claude Code">
```bash
claude mcp add mcpu -e SERIAL_PORT=/dev/ttyACM0 -- npx mcpu-client
```
  </TabItem>
  <TabItem label="3 вҖ” Claude Desktop config">
```json
{
  "mcpServers": {
    "mcpu": {
      "command": "npx",
      "args": ["mcpu-client"],
      "env": { "SERIAL_PORT": "/dev/ttyACM0" }
    }
  }
}
```
  </TabItem>
</Tabs>

## Built-in Tools

Every device running MCP/U exposes these tools automatically:

| Tool | Description |
|------|-------------|
| `list_tools` | Discovery: tools + pin registry |
| `get_info` | Device name, version, platform |
| `gpio_write` | Set digital output HIGH / LOW |
| `gpio_read` | Read digital level |
| `pwm_write` | PWM output (duty 0вҖ“255) |
| `adc_read` | Read ADC value (0вҖ“4095) |
